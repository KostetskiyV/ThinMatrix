# ThinMatrix

## Условие задачи
Разреженная (содержащая много нулей) матрица хранится в форме 3-х объектов (CSR):
- вектор A содержит значения ненулевых элементов;
- вектор JA содержит номера столбцов для элементов вектора A;
- вектор IA, в элементе Nk которого находится номер компонент в A и JA, с которых начинается описание строки Nk матрицы A.

Смоделировать операцию сложения двух матриц:
1. Хранящихся в этой форме, применяя алгоритм работы со сжатой матрицей, с получением результата в той же форме.
2. Хранящихся в стандартной форме, применяя стандартный алгоритм работы с матрицами.
3. Сравнить время выполнения операций и объем памяти при использовании этих 2-х алгоритмов при различном проценте заполнения матриц. 
## Исходные данные: 
Взаимодействие с программой происходит через меню-интерфейс. Пользователь вводит числа (0-4) для выбора действий программы.

1, чтобы ввести матрицы

2, чтобы считать матрицы из файла matrix.txt

3, чтобы вывести матрицу

4, чтобы сложить матрицы

0, чтобы завершить

Для ручного ввода матрицы и для ввода из файла даётся выбор:

1, для обычного формата

2, для координатного формата

0, для отмены

Описание матрицы плотного формата подразумевает:
В первой строке 2 целых числа больше 0, задающих размер матрицы: количество строк  (n) и количество столбцов (m), далее следует n строк по m целых чисел.
Описание матрицы координатного формата подразумевает:
В первой строке 3 целых числа: первые два - больше нуля - задают размер матрицы: количество строк  (n) и количество столбцов (m), третье число (k) указывает количество ненулевых элементов в матрице (k ≤ n*m). Далее следует k строк по 3 числа: первые два - координаты ненулевого элемента, третье – значение.
Формат требует последовательное введение двух матриц - слагаемых.
## Выходные данные:
Для вывода матрицы даётся выбор формата (выводятся только инициализированные матрицы):

1, чтобы вывести матрицы в обычном формате

2, чтобы вывести матрицы в координатном формате

3, чтобы вывести матрицы в формате CSR

0, для отмены

Обычный и координатный формат вывода аналогичен формату ввода.
Формат вывода CSR подразумевает вывод трёх массивов:

- an – содержит значения всех ненулевых элементов в порядке сверху-вниз, справа-налево
- ja – содержит горизотальные индексы всех элементов an
- ia – содержит n+1 элементов – индексы элементов an, ja для каждой из строк матрицы, последняя строка содержит значение – количество ненулевых элементов.

При вводе 0, программа перманентно завершается без каких-либо сообщений.
При сложении матрицы выводится время сложения и затраты памяти на операцию для плотных и CSR матриц.
## Задача, реализуемая программой
Сложение плотных матриц и матриц формата CSR. замеры затрат времени и памяти на операцию.
## Способ обращения к программе
Через терминал (app.exe).
## Возможные аварийные ситуации и ошибки пользователя
Некорректное обращение к меню. исспользование недопустимых символов, некорректные размеры матриц, координаты элементов. Попытка считать матрицу из несуществующего файла matrix.txt. Нехватка памяти для инициализации матрицы. Попытка сложить матрицы не одинакового размера или неинициализированные матрицы.
